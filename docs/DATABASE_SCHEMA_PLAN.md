## Design Principles

- Separate DB, never shared with other ecosystem apps
- JSONB for flexible structured data (annotations, AI output)
- UUIDs as primary keys (portable, Aegis-friendly)
- All tables include `created_at`, `updated_at` timestamps
- Soft deletes via `is_archived` flag where data has relational value
- No polymorphic hacks — prefer explicit foreign keys

## Indexes

```sql
-- Performance indexes for common query patterns
CREATE INDEX idx_cases_submitter ON cases(submitter_id);
CREATE INDEX idx_cases_status ON cases(status);
CREATE INDEX idx_cases_created ON cases(created_at DESC);
CREATE INDEX idx_case_media_case ON case_media(case_id);
CREATE INDEX idx_case_tags_case ON case_tags(case_id);
CREATE INDEX idx_annotations_case ON annotations(case_id);
CREATE INDEX idx_comments_case ON comments(case_id);
CREATE INDEX idx_comments_parent ON comments(parent_id);
CREATE INDEX idx_tags_category ON tags(category);
CREATE INDEX idx_tags_slug ON tags(slug);
```
